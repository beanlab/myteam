# Code Style

Write self-documenting code: break functions up into smaller functions with clear, concise names. 

Names of functions and variables should be helpful and accurate but not be overly context-specific.

Include type hints, but omit the default `-> None` on return-less functions.

Follow **pragmatic engineering**: don't build more than is needed right now.

Follow **incremental complexity**: don't add complexity until it is actually needed. Don't scaffold for future complexity, the future can scaffold itself. Don't add abstractions until they are needed.

Follow **software craftsmanship**: write clean, maintainable, correct code. Don't cut corners.
