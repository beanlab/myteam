## Role Metadata And Delegation Boundaries

Summary:
The discussion opened with role front matter, role descriptions, and
how much implementation detail should live in role text. The group
argued for trigger-focused descriptions that explain when to call a
role without leaking enough detail for other agents to bypass the
intended workflow or tool boundaries.

Decisions:
- Keep role descriptions focused on when to call the role.
- Prefer deterministic workflows over prompt-only delegation control.

Tasks:
- [Unassigned] Continue refining the role creator so it fills out
  role files more effectively.

Open Items:
- How much role metadata belongs in front matter versus markdown?
- How can roles stay clear without leaking implementation intent?

## Workflow Terminal UI

Summary:
A Textual-based terminal UI for workflows was demonstrated. The UI
shows step state, role context, thread or session details, agent
output, and an input area for interaction. The group focused on
resume behavior, reducing low-value visual clutter, and keeping the
UI loosely coupled to the underlying workflow engine.

Decisions:
- Treat the terminal UI as a front end that should stay separate
  from the workflow engine.
- Improve the interface so conversation is more prominent than
  low-level status detail.

Tasks:
- [Unassigned] Improve workflow resume behavior so partial responses
  are not lost on quit.
- [Unassigned] Rework the terminal UI layout to emphasize the step,
  role, and visible conversation.
- [Unassigned] Show an example of the updated YAML format in a future
  meeting.

Open Items:

- What backend interface is needed if multiple workflow engines are
  supported later?

## Domain-Aware Skill Creation

Summary:
The final discussion considered whether skill creation should be
split into generic formatting help and a domain-specific research
process. The concern was that a single creator works well for
structure, but not for subject-matter design in areas like CI/CD or
cloud deployment without prior research.

Decisions:
- Separate general skill formatting concerns from domain discovery
  and research.
- Treat research-first skill design as a process worth exploring.

Tasks:
- [Unassigned] Explore a research-first workflow for building
  subject-specific skills.

Open Items:
- Should domain-specific skill design live in a skill, a role, or a
  workflow?
- What sources should a skill-designer process gather before writing
  the final skill text?
