# Meeting Notes

Date: 2026-04-13 11:00

## Summary

The group discussed role metadata, workflow determinism, and a new
terminal UI for running `myteam` workflows. The main theme was that
trigger-focused descriptions and explicit workflow steps seem more
reliable than relying on prompt tuning for correct delegation.

## Main Points

- Newer roles appear to support front matter similar to skills and
  rules.
- Role descriptions should stay focused on when to call the role,
  not on revealing implementation details.
- Deterministic workflows were favored over trying to perfect
  delegation behavior through prompt wording alone.
- A Textual-based terminal UI was demonstrated for running and
  interacting with workflow steps.
- The UI already supports stepping through runs and resuming them,
  but session-state persistence is incomplete.
- The team discussed separating UI concerns from the underlying
  workflow engine so the backend can be swapped later.
- Skill creation may need to be split into general formatting help
  and domain-specific research or design support.

## Decisions

- Keep role descriptions trigger focused.
- Continue exploring deterministic workflow control rather than
  over-optimizing delegation prompts.
- Treat the terminal UI as a front end that should stay loosely
  coupled to the workflow engine.
