# Meeting Notes

Date: 2026-04-06 11:05

Summary:
The discussion focused on improving the meeting transcription
workflow and evaluating how `myteam` workflows should manage
multi-step agent conversations. The group compared app server and
`codex exec` approaches, with particular attention to explicit
conversation completion, tool handling, and structured output.

Main points:
- Meeting notes and transcript retrieval were useful in practice.
- A desired workflow is a one-key recording shortcut that saves
  low-bitrate speech audio in the right location and auto-ingests it.
- `myteam` now includes workflow scaffolding and commands for viewing
  and starting workflows.
- App server support is promising for thread control and event
  routing, but completion behavior remains unclear.
- `codex exec` was seen as stronger for explicit termination, but it
  would require user communication to happen through tools.

Decisions:
- Keep agent workflow stage transitions explicit for now rather than
  relying on the model to infer when a conversation is complete.
- Continue exploring workflow support in `myteam` while treating
  app server and `codex exec` as competing implementation paths.

Tasks:
- Design a shortcut-driven recording workflow that captures
  low-bitrate speech audio, stores it in the correct folders, and
  triggers ingestion automatically.
- (David) Look into whether app server can emit JSON events in the
  needed form.
- Investigate how Codex app server handles tool calls in practice.
- Review how workflow completion output is produced and where JSON
  formatting happens in the current `myteam` implementation.
- Explore whether `codex exec` can expose cleaner structured events or
  JSON-like output in addition to its final stdout result.

Open questions:
- Does app server handle Codex tool calls directly, or must wrappers
  implement that behavior themselves?
- What is the most reliable signal that an agent is finished?
- Can app server provide the same kind of structured event stream that
  makes it attractive, while still supporting explicit completion?
- Should workflow completion be done through a dedicated tool or
  follow-up LLM call that turns an ongoing conversation into the
  required final output format?
