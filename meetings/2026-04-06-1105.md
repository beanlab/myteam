## Workflow Engine Architecture And Completion Control

Summary:
Most of the meeting focused on whether `myteam` workflows should run
through Codex app server, `codex exec`, or some hybrid approach. The
discussion centered on thread control, structured outputs, user
interaction, slash commands, and whether workflow transitions should
remain explicitly human-controlled for now.

- Keep workflow stage transitions explicit instead of trusting the
  model to infer completion.
- Continue exploring both app server and `codex exec` as competing
  implementation paths.

  transitions under client control.
