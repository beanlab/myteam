# Meeting Notes

Date: 2026-04-08 11:00

Summary:
The group reviewed `myteam` workflow design, especially whether
workflow steps should be framed as interactive sessions managed
through app server. The discussion emphasized standardizing thread
handling, making workflow dependencies explicit, improving role and
skill creation tooling, and keeping most reusable helpers in roster
rather than moving them into built-ins too quickly.

Main points:
- Use `interactive` rather than `automatic` as the workflow field
  name because it better matches the intended behavior.
- Prefer one thread-management path through app server if it can
  support both conversational and effectively non-interactive runs.
- Rename workflow step `additional context` to `input`.
- Model workflow dependencies explicitly through referenced outputs
  rather than through file-system side effects or implied ordering.
- Consider keyed workflow steps with dependency-based ordering rather
  than list-only step definitions.
- Sequential disclosure and resumable sessions are important workflow
  design goals.
- `myteam new role` and `myteam new skill` should better support
  quick creation and direct editing of generated files.
- Reusable creator helpers should stay in roster for now while they
  are still evolving.
- `myteam` is being positioned as a flexible harness platform for
  less standardized work rather than a rigid built-in workflow suite.

Decisions:
- Use `interactive` as the preferred term for workflow step behavior.
- Standardize on explicit workflow dependencies where possible.
- Treat roles and skills as standalone documents; start them at
  heading level one.
- Keep the built-in surface small and continue incubating most shared
  skills in roster.

- Investigate keyed workflow step definitions with dependency-based

- Rename workflow step `additional context` to `input`.
- Improve `myteam new role` and `myteam new skill` so they print

- Keep working on creator tooling for roles and skills in roster.

  supports dependency-driven execution?
