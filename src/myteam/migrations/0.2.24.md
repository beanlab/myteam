## 0.2.24 migration

Version `0.2.24` fixes role and skill loader output so listed tool paths stay rooted at the project tree.

### What changed

- Role and skill loaders now resolve listed tools from the project root instead of from the copied
  `.myteam/` tree.
- Tool paths remain executable from the repository root without dropping the `.myteam/` prefix.
- Existing generated loaders that already use the current packaged templates do not need edits.

### How to migrate an existing `.myteam` folder

1. Review any copied root, role, and skill `load.py` files for older loader logic.
2. If a loader still passes the `.myteam` tree itself to `list_tools(...)`, update it so tools are
   listed from the project root instead.
3. If you have customized loader wrappers around the packaged built-in skills, refresh them so they
   match the current packaged templates and path resolution behavior.
4. After any approved loader updates are complete, update `.myteam/.myteam-version` to `0.2.24`.
