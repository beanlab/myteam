## 0.2.11 migration

Version `0.2.11` adds packaged workflow execution support for project-local agent trees.

### What changed

- Added `myteam start <workflow>` to load and run workflow definitions from the active local agent
  root such as `.myteam/` or a custom `--prefix` path.
- Workflow definitions are YAML files stored directly in the active agent root and may use either
  `.yaml` or `.yml` extensions.
- Workflow files are resolved by path name relative to the active agent root. A request such as
  `myteam start reports/daily` matches `reports/daily.yaml` or `reports/daily.yml`.
- Workflow files must contain a top-level mapping of step names to step definitions. Each step must
  define `prompt` and `output`, and may also define `input` and `agent`.

### How to migrate an existing `.myteam` folder

There is no required migration for existing `.myteam` folders in this release.

Recommend the user look at the README.md for additional information on the experimental workflows feature.

For this migration, simply update `.myteam/.myteam-version` to `0.2.11`.
