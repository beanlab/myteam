## 0.2.13 migration

Version `0.2.13` refactors experimental workflow execution around a terminal result channel.

### What changed

- Added `myteam workflow-result` so a workflow step can return its structured result over a private
  result channel instead of printing terminal completion markers.
- Workflow step prompts now instruct the agent to call `myteam workflow-result` exactly once with
  the final payload that matches the authored output schema.
- Workflow execution now waits for an out-of-band structured result rather than scraping
  `OBJECTIVE_COMPLETE` JSON from terminal transcript output.
- The workflow runtime was split into focused workflow, agent-backend, and terminal transport
  modules, while preserving the authored workflow file shape.
- Authored workflow YAML still uses the same required `prompt` and `output` fields with optional
  `input` and `agent`.

### How to migrate an existing `.myteam` folder

Most existing `.myteam` folders do not need file changes for this release.

If you have copied or customized workflow-specific role instructions, helper scripts, or wrapper
agents that still tell the agent to print `OBJECTIVE_COMPLETE` JSON in the terminal, update that
guidance to call `myteam workflow-result` exactly once with the final payload instead.

If you only author workflow YAML files and rely on the packaged workflow prompt, there is no
required content migration beyond updating the tracked version.

For this migration, update `.myteam/.myteam-version` to `0.2.13` after any approved workflow
instruction updates are complete.
