## 0.2.10 migration

Version `0.2.10` updates the default loader presentation order and makes packaged changelog notes
available from installed `myteam` releases.

### What changed

- Newly generated role loaders now present skills first, then tools, and roles last.
- Newly generated skill loaders now present nested skills first, then tools, and roles last.
- Packaged release notes now ship inside the installed `myteam` package so `builtins/changelog`
  works after installation and not only from a source checkout.
- The packaged explanation text for roles, skills, and tools was refreshed; loaders that call the
  helper functions automatically pick up that prose from the installed package.

### How to migrate an existing `.myteam` folder

1. Review your root, role, and skill `load.py` files for copied template logic.
2. If a loader currently prints or lists roles before skills and tools, reorder it so skills are
   shown first, tools second, and roles last.
3. If a customized built-in maintenance loader was copied into project files, update it to use
   `get_active_myteam_root()` so it honors the active project root selected by `--prefix`.
4. Do not copy packaged changelog files into your project tree. The installed `myteam` package now
   carries the changelog data needed by `builtins/changelog`.
5. After the approved loader updates are complete, update `.myteam/.myteam-version` to `0.2.10`.
