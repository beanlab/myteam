## 0.2.6 migration

Version `0.2.6` adds tracked `.myteam` versions and built-in upgrade skills for older projects.

### What changed

- New `.myteam/.myteam-version` metadata records which `myteam` release most recently initialized
  or refreshed the local agent tree.
- New built-in maintenance skills are scaffolded under `.myteam/myteam/`, including
  `myteam/migrate` and `myteam/changelog`.
- The default root `load.py` can now alert the agent when the installed `myteam` version is newer
  than the tracked `.myteam` version.
- Migration guidance is now meant to be reviewed by an agent and applied as project-specific edits,
  rather than by a dedicated CLI migration command.

### How to migrate an existing `.myteam` folder

1. Review these migration notes and the matching changelog entries for versions newer than your
   tracked `.myteam` version.
2. Add the built-in maintenance skill subtree under `.myteam/myteam/` if your project does not
   already have it, including `myteam/migrate` and `myteam/changelog`.
3. If your project uses the default root `.myteam/load.py`, update it so it can surface version
   upgrade notices when appropriate. If your project uses a customized root loader, adapt that
   custom logic instead of overwriting it blindly.
4. Once the approved project-specific updates are complete, update `.myteam/.myteam-version` to the
   installed `myteam` release.
