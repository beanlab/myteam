## 0.2.12 migration

Version `0.2.12` improves experimental workflow step execution reliability.

### What changed

- Workflow step prompts are now injected after the configured agent session emits output, which
  gives interactive agents time to initialize their terminal input handling.
- The PTY wrapper now submits injected input with the same separated key sequence used for graceful
  shutdown commands.
- Workflow completion detection now tolerates terminal control sequences around wrapped JSON
  completion payloads.
- Workflow agents are asked to exit only once after a valid completion payload is accepted.

### How to migrate an existing `.myteam` folder

There is no required migration for existing `.myteam` folders in this release.

For this migration, simply update `.myteam/.myteam-version` to `0.2.12`.
